# TaskMaster PRO (Full Stack Task Management System)

An advanced, full-stack task management application showcasing a structured, clean architecture across a Python FastAPI backend and a Next.js React frontend.

## 🚀 Features
- **JWT Authentication**: Short-lived access tokens + long-lived refresh tokens seamlessly integrated via Axios interceptors.
- **Task Management**: Full CRUD operations + archiving, status, and priority tracking.
- **Task History**: Automatic historical tracking of every status and detail change.
- **Analytics Dashboard**: Interactive charts displaying tasks completed over time, priority distribution, and custom productivity score logic using Recharts.

---

## 🛠️ Tech Stack
**Backend**: FastAPI, SQLAlchemy, Alembic (Migrations), SQLite (Default) or PostgreSQL (Optional via Docker Compose), passlib (bcrypt), python-jose (JWT).  
**Frontend**: Next.js (App Router), React, Tailwind CSS, shadcn/ui, Recharts, Axios, js-cookie.

---

## 🏗️ System Architecture & Design
### Backend Structure
- `app/api/v1/endpoints`: Route definitions separated by logical domains (auth, tasks, analytics)
- `app/core`: Core configuration, security utilities, and database engine setup.
- `app/models`: SQLAlchemy Declarative Models.
- `app/schemas`: Pydantic models for strict request validation and response serialization.

### Security (JWT Flow)
1. User logs in -> Returns `access_token` and `refresh_token`.
2. Frontend Axios interceptor automatically attaches the `access_token` as a Bearer token.
3. If the backend returns a `401 Unauthorized` (Token expired), the response interceptor catches it, automatically hits `/auth/refresh` using the `refresh_token` stored in Cookies, updates the tokens, and seamlessly replays the failed request without bothering the user.

### Analytics Logic Calculation
- **Productivity Score**: Calculated as `10 Base Points per Completed Task + 5 Bonus points for completing High Priority Tasks`.
- **Average Completion Time**: Computes the `updated_at - created_at` duration for all completed tasks and converts to hours.
- **Tasks per Day**: Aggregation using the `updated_at` (or `created_at`) grouping for visually rendering on a timeline.

## 🏁 Setup Instructions

### 1. Database (SQLite is default, PostgreSQL Optional)
A `docker-compose.yml` file is provided if you wish to use PostgreSQL. 
By default, the backend operates on SQLite (`task_management.db`) for immediate, zero-dependency testing. 

### 2. Backend Setup
```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt # (or manually install via setup step)
```

**Environment Variables**: A `.env` file is generated by default (or can be configured matching `app/config/settings.py`).
`SECRET_KEY`, `DATABASE_URL`

**Run Migrations**:
```bash
export PYTHONPATH=$PWD
alembic upgrade head
```

**Start the Server**:
```bash
fastapi dev app/main.py   # Or uvicorn app.main:app --reload
```
View Swagger API documentation at: `http://localhost:8000/docs`

### 3. Frontend Setup
```bash
cd frontend
npm install
npm run dev
```
Access the application at: `http://localhost:3000`
